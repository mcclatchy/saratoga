AMP components

AMP pages at the core are just web pages with some components and restrictions that are intended to make the web faster and pages more portable. Pages do have the restriction of 50K of CSS, but AMP doesn't dictate what we can do with that 50K. Since AMP pages are only sent to mobile devices, we actually have better support of newer native technologies. There is also no restriction on HTML except for a couple of elements, so most of the design for the new story pages can be done here as well.

For the custom elements that are shipped with AMP, there is a concept called layout that is worth learning. In several cases it is required, or different options require different attributes be set. The purpose is to be able to determine the height of elements like images and ads before the content loads, preventing content from jumping around. It's quirky, to say the least, but we're stuck with it.

Additional components not supported by Relay Media

The list of components that we might potentially use is very long and recreating each one here would be overkill. I highly recommend all departments at least skim the functionality of every component on ampproject.org. I'm including a list of those I think might help us the most. If anyone would like to see a component in action I will make a sample here for us to investigate.

Dynamic content

<amp-bind> reference

This component is the backbone of interaction in AMP pages. AMP doesn’t allow developers to use JavaScript as a method to interact with the page directly, but using <amp-bind> we can still perform some basic tasks like doing something on a click.

<amp-list> reference

This component fetches a JSON file remotely and can run the data through a mustache template. Although it’s probably not something the be included in a story body by reporters, it may come in useful when building the bottom of the story page or in personalization.”

Note: There is a cousin component called <amp-live-list> that can refresh without reloading the page. Our time on site is probably short enough this doesn’t matter though.

<amp-selector> reference

A control that allows actively selected state in a list of children. This is very similar to the component in the Polymer project if you’re familiar with that. All it does is keep track of what child has been selected and provides a visual class to show the user what they have selected. Pretty boring on its own.

Couple this with data binding allowed in AMP, and you can do some fun things. Take a gallery using <amp-carousel> for example. If we add a thumbnail strip underneath declared with <amp-selector>, when a user clicks a thumbnail the component fires an event that the carousel can use to show the corresponding image.

Layout

<amp-accordion> reference

A standard accordion. It can be used to toggle elements in a list, or for concepts like a “show more” button.

<amp-carousel> reference

A basic photo gallery. The most direct way of using it is to load it up with a stack of <amp-img> tags. The component provides the arrows and interactive elements. There are several different types of display all packaged into one element, so I recommend reading this one thoroughly if we want to use it.

Technically, the carousel will work with any div placed inside it. We could think out of the box for use cases, but generally speaking carousels should be doused in kerosene and ignited for warmth.

<amp-lightbox> reference

A basic lightbox for any content. If we want to use this, I should spend some time figuring out what we can style. There is also an <amp-image-lightbox> and an <amp-lightbox-gallery> component for more specific use cases.

<amp-position-observer> reference

I recommend reading through this one a couple of times for ideas. This is an under-the-hood component that can be linked to others to produce scroll effects.

<amp-sidebar> reference

Relay Media uses this for the hamburger menu functionality, and that’s what it’s for mostly. It can be linked up to other elements on the page, like the actual hamburger icon, to open and close.

Presentation

<amp-animation> reference

I really didn’t like Polymer’s implementation of animation and this is basically a copy of that concept. The Polymer team eventually bailed on the whole idea and told developers to just use a JS/CSS one-two-punch combo, but AMP is stuck with this. It can do several things, but one of the cooler ones is to trigger an animation when an element hits the viewport.

For each animation we want to try to do I’ll check against this documentation.

Social

<amp-social-share> reference

Your basic row of social media icons to share the story #usethephone. This component is actually very easy to use and is well built.
